TAO / Bittensor

Source of truth remains Markdown. This page is the readable GitHub Pages presentation layer.

Overview
  • Decentralized AI incentive network.
  • Native token: TAO.
  • Core design: subnets compete for emissions by providing machine intelligence services such as inference, training, data, agents, search, compute, and specialized ML tasks.
  • Major upgrade: dTAO introduced subnet alpha tokens and market-driven subnet allocation dynamics.

Risks

  • Bittensor is not one product; it is an incentive layer for many subnet businesses.
  • TAO valuation increasingly depends on subnet quality, alpha-token dynamics, and real external demand.
  • Avoid evaluating only TAO spot price; subnet emissions, alpha performance, and validator behavior are critical.

Watch triggers

  • Major protocol upgrade or dTAO parameter change.
  • Sharp emission rotation among top subnets.
  • Large validator stake movement or VTrust degradation.
  • Security issue in Subtensor, wallets, staking, bridges, or subnet code.
  • Regulatory action targeting TAO, validators, or AI/compute marketplaces.
  • Network congestion, chain halt, or validator/miner coordination failure.
  • Subnet revenue narratives diverging from on-chain activity.
