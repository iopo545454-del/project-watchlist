source of truth

Canonical files

Edit these first; generated pages should follow them.
  1. projects/*.mddossiers

    Primary research files: overview, thesis, catalysts, sources, open questions, monitoring notes, unverified watch items.

  2. index.json / index.csvroot index

    Machine-readable project metadata used by scripts and future agents.

  3. docs/data/index.jsondashboard data

    Powers the project cards on the GitHub Pages dashboard.

  4. docs/data/project-changelog.jsonlatest feed

    Powers dashboard changelog and individual project Latest panels.

  5. docs/data/scan-debug.jsonoperator log

    Powers the scan-debug page: run status, decision trail, errors, next checks.

  6. docs/projects/*.htmlgenerated presentation

    Published project pages. Do not treat them as canonical; regenerate/update them from Markdown/data.

modify

Best way to change something

Pick the workflow by intent.
  1. Add a new projectpublic-safe

    Use the dashboard Add project button or open a GitHub issue with project name, token/contract, main X, and context. The 4h Hermes job processes intake issues, creates the dossier/page, and runs initial review.

  2. Correct project factsdurable edit

    Edit the relevant projects/<slug>.md section first, then update indexes/changelog/generated HTML so the live site matches.

  3. Add sources or watch triggersmonitoring edit

    Add them to the dossier source table / monitoring notes. Good source entries include what the source is and when/why Hermes should check it.

  4. Change scan behavioragent rules

    Patch AGENTS.md for durable repo policy. Patch the 4h cron prompt only for scheduler-specific behavior. Repo rules are easier for future agents to reference.

  5. Change the UIpresentation edit

    Modify docs/index.html, docs/app.js, and docs/style.css. Keep the data contracts stable or update the JSON producers at the same time.

automation

How Hermes is managing it

Recurring scanner plus GitHub Pages.
  • A Hermes cron job named Project watchlist research scans — 4h runs every four hours in /root/project-watchlist-github.
  • It scans tracked project sources, X accounts, intake issues, weak signals, catalysts, and material updates.
  • It updates Markdown/data/HTML, validates JSON/diffs, commits, and pushes to GitHub when there are real changes.
  • GitHub Pages serves the dashboard from /docs. CDN lag can make the public site trail raw GitHub briefly.
  • Discord alerts should stay quiet unless there is material news; routine scan evidence belongs in the dashboard/debug data.
